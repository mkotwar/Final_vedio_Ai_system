# File Structure

This repository keeps production code, formal tests, and long-lived project
assets only. One-off probes, root-level debug scripts, generated reports, local
databases, and temporary outputs should not be committed.

## Required Source

```text
app/              FastAPI backend, schemas, services, and API routers
config/           Runtime configuration data used by services
frontend/         Static dashboard served by FastAPI
tests/            Formal pytest suite
docs/             Project documentation and roadmap
scripts/          Reusable maintenance scripts only
validation/       Stable validation datasets and audit fixtures
requirements.txt  Python dependency list
README.md         Setup and usage guide
AGENTS.md         Repository operating instructions
yolo11m.pt        Local YOLO detector weight used by ObjectDetector
```

## Runtime Data

```text
data/videos/
data/frames/
data/metadata/
data/events/
data/logs/
data/thumbnails/
data/detections/
data/qdrant_db/
```

These folders are generated by the application. They may exist locally for
development or investigations, but they are not source code.

## Removed Clutter

The cleanup removed root-level audit scripts, duplicate loose test scripts,
debug outputs, generated reports, local Qdrant cache files, `scratch/`, and
`experiments/`. New tests should go under `tests/`; reusable operational tools
should go under `scripts/`.
